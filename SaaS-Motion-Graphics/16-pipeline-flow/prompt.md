# Prompt — 16-pipeline-flow

The original prompt used to generate this motion graphic:

A white canvas. Seven nodes arranged in a horizontal line — the stages of a deployment pipeline. Each node is a circle. The labels beneath them read: Commit. Build. Test. Scan. Stage. Approve. Deploy.
They all appear simultaneously, springing in with staggered timing so tight it reads almost as one motion — left to right, 4 frames between each. Connector lines draw between all six gaps simultaneously, all drawing at the same speed, all arriving at their destinations together. The pipeline is assembled in one coordinated motion rather than sequentially.
A single large dot — larger than a traveling dot, more like a token — launches from Commit and begins traveling the pipeline. It moves at a deliberate pace, slower than expected. It reaches Build — which turns blue. Reaches Test — which turns blue. Reaches Scan — which turns blue. Reaches Stage — which turns blue.
It reaches Approve — and stops. The node pulses amber. The token sits there. Three seconds pass. Then the node turns green and the token continues to Deploy — which turns green and holds green. The token disappears. All nodes hold their colors. The pipeline is done.
