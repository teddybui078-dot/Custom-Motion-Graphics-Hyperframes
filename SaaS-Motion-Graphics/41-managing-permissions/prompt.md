# Prompt — 41-managing-permissions

The original prompt used to generate this motion graphic:

Managing Permissions
Duration: 18 seconds

A team settings page on white. A members list — eight rows, each showing an avatar, name, email, and a role badge: Admin, Editor, or Viewer. A cursor clicks the role badge on the fourth member — a dropdown opens showing all available roles. The cursor clicks Editor. The badge updates — a fast color crossfade from one color to another. The cursor clicks the role badge on the sixth member. Changes to Viewer. Clicks the seventh — changes to Admin. Three role changes, three badge updates, 6 seconds.
The cursor clicks a "Permissions" tab at the top of the settings page. A permissions matrix fills the frame — rows of features, columns of roles, a grid of toggle switches. Several toggles are on, several off. The cursor clicks a toggle in the Viewer column for a specific feature — it flips off with a spring. Two more toggles adjusted. The cursor clicks Save Changes. A toast confirms. The matrix holds — the new permission state visible, the system updated. The team accesses only what they should.
