# Prompt — 46-prompting-lovable

The original prompt used to generate this motion graphic:

Prompting Lovable Duration: 30 seconds

A browser frame fades in on a white canvas. Lovable's interface fills the frame — clean, minimal, the familiar split layout: a chat input panel on the left, an empty preview panel on the right. The Lovable logo sits in the top left. The preview panel shows a blank white canvas with a faint grid — nothing built yet. The interface is waiting.

A cursor clicks into the chat input field at the bottom of the left panel. A prompt types itself in — deliberate, specific, the way a real user who knows what they want types:

"Build me a SaaS landing page with a hero section, feature grid, pricing table with three tiers, and a footer. Use a dark navy and electric blue color scheme. Make it modern and clean."

The cursor clicks the Send button. The message bubble locks into the chat panel — the user's prompt sitting there, sent, real. A beat of stillness. Then Lovable starts thinking.

A typing indicator appears in the chat panel — three dots pulsing. Then Lovable's first response message materializes word by word, fast but readable:

"Got it. Building your SaaS landing page now. Starting with the project structure..."

The left panel shifts — below the chat, a file tree materializes. It builds itself top to bottom, one file appearing at a time with a tight stagger, each filename fading in as if being created in real time:

📁 src 📄 index.html 📄 App.tsx 📄 main.tsx 📁 components 📄 Hero.tsx 📄 Features.tsx 📄 Pricing.tsx 📄 Footer.tsx 📁 lib 📄 pricing.ts 📄 utils.ts 📄 tailwind.config.ts

Each file appears with a 4-frame stagger, the folder structure assembling itself naturally, indentation showing hierarchy. As each file appears, a small green dot pulses beside it — the AI is writing it.

Simultaneously, a code panel slides open in the left panel below the file tree — a dark-background code editor, syntax highlighted. Code begins streaming into it — fast, the way AI writes code, line after line appearing in rapid succession. The first file being written is index.html — the DOCTYPE declaration appears, then the html tag, then head with meta tags, then the body. The code streams too fast to read line by line but slow enough that the structure is visible — indentation, closing tags, the shape of real HTML. A filename tab at the top of the code panel shows index.html — active, being written.

The tab switches — App.tsx appears in the tab. TypeScript code streams in — import statements in blue, component definitions in white, JSX in amber. The syntax highlighting makes it readable even at speed. The tab switches again — Pricing.tsx. A pricing component taking shape, the tier structure visible in the code as it streams — three objects, three price points, three feature arrays building themselves out line by line.

The right preview panel activates. It is no longer blank — it begins rendering. Not all at once. Section by section, top to bottom, the page assembles itself in the preview as the code is written:

The navbar appears first — dark navy background, logo placeholder on the left, navigation links on the right, a blue CTA button. It drops in from the top of the preview with a subtle fade.

The hero section renders below it — a large headline in white, a subheadline in lighter gray, two CTA buttons side by side, and a product UI mockup on the right side of the hero. The hero fades in as a unit.

The feature grid appears — a 3×2 grid of feature cards, each with an icon, a title, and a description. The cards appear one at a time in a left-to-right, top-to-bottom cascade, each card fading in with a 6-frame stagger.

The pricing table renders — three cards side by side, the center card slightly elevated with a "Most Popular" badge. Each card shows a plan name, a price, a feature list with checkmarks, and a CTA button. The three cards appear simultaneously, all three fading in together.

The footer renders last — a dark navy band across the bottom, four columns of links, a copyright line. It fades in from the bottom.

The preview panel now shows a complete, designed, functional-looking SaaS landing page. Dark navy background, electric blue accents, clean typography, the exact brief the user typed. It is fully scrollable — the preview panel scrolls automatically, slowly revealing the full page length from top to bottom, the user seeing everything that was built.

A final Lovable chat message appears in the left panel, word by word:

"Your landing page is ready. All sections built — hero, features, pricing, and footer. Click any section in the preview to edit."

The cursor moves from the chat panel to the preview panel and clicks the hero headline. An inline edit handle appears around it — a blue border, resize handles at the corners. The cursor double-clicks the headline text. It becomes editable — a blinking cursor appears inside the text. The user types two new words, replacing the placeholder headline. The preview updates live — the new headline visible in the rendered page instantly as it's typed.

The cursor clicks outside the edit handle. The page holds — complete, customized, alive. The file tree on the left shows all files with solid green dots — every file written, every component built. The code panel shows the final state of the last edited file.

One beat of complete stillness. The browser frame, the file tree, the code panel, the live preview — all holding together. The whole scene visible at once. Cut to white.
